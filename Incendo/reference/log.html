<log> element

Used within the <display> or the <print> element to display or print a log.

The logs must first be declared using the <logdcl> element. The log data is recorded using the <logrec> element.

The required attributes name and layout define which of the logs is to be displayed or printed and which of the layouts (rendering patterns) is to be used.

The optional type attribute specifies the order in which the log records should appear.

Note that your service must have the appropriate read permissions to access the log.

Parent elements:

baddata dd display div form layout li pinentry print prompt td th

Attributes

NameTypeDescription
namexs:NMTOKEN

Required attribute.

Specifies the name of the log that should be displayed or printed. The attribute value should correspond to the name attribute of the corresponding <logdcl> element.

layoutxs:NMTOKEN

Required attribute.

Specifies the name of the layout (rendering pattern) for the log records. The value of this attribute should correspond to the name attribute of one of the child <layout> elements of the corresponding <logdcl> element.

typexs:token

Optional attribute defining the order in which the log records are to be shown.
The value should be one of the following:

  • normal - Most recent records should be shown closer to the end (bottom) of the log. The oldest records will be at the start of the log.
  • reverse - Most recent records should be shown at the start of the log.

Default value is normal .

Child elements:

No child elements.
