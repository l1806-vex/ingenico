<run> element

Not yet implemented.

The <run> element is used for calling of external application (plug-in) or service.

The order and the source of input parameters are specified by the child <in> elements.

After the application call, the result is passed using the output parameters, specified by the child <out> elements.

Parent elements:

screen

Attributes

NameTypeDescription
nameValref

Specifies the id of the plug-in or a service that should be called.

This attribute can be either a value string or a reference to the variable that contains the value.

If variable reference is used, the name of the variable must be prefixed by tmlvar: i.e. tmlvar:variable.name

typexs:token

Specifies the type of the application.
The value should be one of the following:

  • dll
  • app
  • vas

Default value is app .

passwordValref

The password for service authentication. (by default is used as blank for no authentication).

This attribute can be either a value string or a reference to the variable that contains the value.

If variable reference is used, the name of the variable must be prefixed by tmlvar: i.e. tmlvar:variable.name

Default value is .

Child elements:

[in* -> out*]
